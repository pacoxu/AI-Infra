# Multi-Cluster Use Cases Diagram Translation

This diagram from KubeCon North America 2025 SIG Multi-Cluster presentation shows
common reasons why organizations run multi-clusters.

## Original Image
Source: https://github.com/user-attachments/assets/5f42f1bb-2d38-4f22-880a-a9512a502983

## Text Translation (English → Chinese)

### Title
- "Why organizations run multi clusters / Use case" → "组织为何运行多集群 / 使用场景"
- "Qualitative highlights" → "定性要点"

### Y-Axis Labels
- "Most common" → "最常见"
- "Common" → "常见"

### Use Case Boxes (Top to Bottom)

1. **Environment** (Blue box)
   - Original: "Development, Staging, Production = Test deployment safely, breaking live systems"
   - Chinese: "环境 — 开发、预发、生产 = 安全测试部署，避免破坏生产系统"

2. **Region** (Pink box)
   - Original: "Cluster/region = high performance, Geo redundancy, better service availability"
   - Chinese: "区域 — 集群/区域 = 高性能、地理冗余、更好的服务可用性"

3. **Business Unit** (Purple box)
   - Original: "Cluster / business unit = Team ownership, independence scaling and updates, easy compliance"
   - Chinese: "业务单元 — 集群/业务单元 = 团队所有权、独立扩展和更新、易于合规"

4. **Isolation** (Light blue box)
   - Original: "Other isolation dimensions e.g. Cluster / Workload = better security and management"
   - Chinese: "隔离 — 其他隔离维度，例如 集群/工作负载 = 更好的安全性和管理"

5. **Infrastructure Provider** (Coral box)
   - Original: "(Run clusters) Single or Multi cloud/ on prem/ Edge locations = Flexible deployment, Independence from vendor lock-in"
   - Chinese: "基础设施提供商 — (运行集群) 单云或多云/本地/边缘位置 = 灵活部署、避免供应商锁定"

### Additional Elements
- "KubeCon North America 2025" logo (top right)
- "CloudNativeCon" logo (top right)
- Orange arrow on left indicating spectrum from "Most common" to "Common"
